title: "Quickstart"
description: "Make your first Suno API call in under ten minutes."
---

<Steps>
  <Step title="Obtain your Suno cookie">
    Suno API authenticates with Suno using your browser session cookie.

    1. Open [suno.com/create](https://suno.com/create) in your browser.
    2. Press `F12` (or right-click → **Inspect**) to open Developer Tools.
    3. Go to the **Network** tab and refresh the page.
    4. In the filter box, type `__clerk_api_version` to find the Clerk authentication request.
    5. Click on that request, then open the **Headers** tab.
    6. Scroll to the **Request Headers** section, find **Cookie**, and copy the entire value.

    <Warning>
      The cookie contains your session credentials. Treat it like a password — do not commit it to version control or share it publicly.
    </Warning>
  </Step>

  <Step title="Get a 2Captcha API key">
    Suno requires an hCaptcha challenge on every generation. Suno API solves it automatically using [2Captcha](https://2captcha.com).

    1. [Register](https://2captcha.com/auth/register?userType=customer) a new 2Captcha account.
    2. [Top up](https://2captcha.com/pay) your balance (a small amount covers many requests).
    3. Copy your API key from the [recognition page](https://2captcha.com/enterpage#recognition).
  </Step>

  <Step title="Clone the repository and install dependencies">
    ```bash
    git clone https://github.com/gcui-art/suno-api.git
    cd suno-api
    npm install
    ```
  </Step>

  <Step title="Create your .env file">
    Copy the example file and fill in your values:

    ```bash
    cp .env.example .env
    ```

    Edit `.env`:

    ```bash
    SUNO_COOKIE=<paste the full cookie string you copied in step 1>
    TWOCAPTCHA_KEY=<your 2Captcha API key>
    BROWSER=chromium
    BROWSER_GHOST_CURSOR=false
    BROWSER_LOCALE=en
    BROWSER_HEADLESS=true
    ```

    <Tip>
      Use `BROWSER_LOCALE=en` or `BROWSER_LOCALE=ru` — these two locales have the highest worker count on 2Captcha, which means faster CAPTCHA solving.
    </Tip>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    The server starts on `http://localhost:3000`.
  </Step>

  <Step title="Verify the server is running">
    Call the quota endpoint to confirm your cookie is valid and the server is healthy:

    ```bash
    curl http://localhost:3000/api/get_limit
    ```

    Expected response:

    ```json
    {
      "credits_left": 50,
      "period": "day",
      "monthly_limit": 50,
      "monthly_usage": 50
    }
    ```

    If you see this response, the server is connected to Suno and your session is active.
  </Step>

  <Step title="Generate your first song">
    Send a prompt to `/api/generate`. The endpoint returns immediately with two clip objects (Suno always generates two variations). The `status` field will be `"submitted"` or `"queued"` — audio is not ready yet.

    <CodeGroup>
      ```bash curl
      curl -X POST http://localhost:3000/api/generate \
        -H "Content-Type: application/json" \
        -d '{
          "prompt": "A popular heavy metal song about war, sung by a deep-voiced male singer, slowly and melodiously. The lyrics depict the sorrow of people after the war.",
          "make_instrumental": false,
          "wait_audio": false
        }'
      ```

      ```python Python
      import requests

      base_url = 'http://localhost:3000'

      def generate_audio_by_prompt(payload):
          url = f"{base_url}/api/generate"
          response = requests.post(url, json=payload, headers={'Content-Type': 'application/json'})
          return response.json()

      data = generate_audio_by_prompt({
          "prompt": "A popular heavy metal song about war, sung by a deep-voiced male singer, slowly and melodiously. The lyrics depict the sorrow of people after the war.",
          "make_instrumental": False,
          "wait_audio": False
      })

      print(data)
      ```

      ```javascript JavaScript
      const axios = require("axios");

      const baseUrl = "http://localhost:3000";

      async function generateAudioByPrompt(payload) {
        const url = `${baseUrl}/api/generate`;
        const response = await axios.post(url, payload, {
          headers: { "Content-Type": "application/json" },
        });
        return response.data;
      }

      const data = await generateAudioByPrompt({
        prompt:
          "A popular heavy metal song about war, sung by a deep-voiced male singer, slowly and melodiously. The lyrics depict the sorrow of people after the war.",
        make_instrumental: false,
        wait_audio: false,
      });

      console.log(data);
      ```
    </CodeGroup>
  </Step>

  <Step title="Poll for completion">
    Take the `id` values from the generate response and poll `/api/get` until `status` becomes `"streaming"` or `"complete"`. At that point `audio_url` contains a playable link.

    <CodeGroup>
      ```bash curl
      # Replace IDs with the actual values from the generate response
      curl "http://localhost:3000/api/get?ids=<id1>,<id2>"
      ```

      ```python Python
      import time
      import requests

      base_url = 'http://localhost:3000'

      def get_audio_information(audio_ids):
          url = f"{base_url}/api/get?ids={audio_ids}"
          response = requests.get(url)
          return response.json()

      # ids from the generate response
      ids = f"{data[0]['id']},{data[1]['id']}"
      print(f"ids: {ids}")

      for _ in range(60):
          result = get_audio_information(ids)
          if result[0]["status"] == 'streaming':
              print(f"{result[0]['id']} ==> {result[0]['audio_url']}")
              print(f"{result[1]['id']} ==> {result[1]['audio_url']}")
              break
          time.sleep(5)
      ```

      ```javascript JavaScript
      const axios = require("axios");

      const baseUrl = "http://localhost:3000";

      async function getAudioInformation(audioIds) {
        const url = `${baseUrl}/api/get?ids=${audioIds}`;
        const response = await axios.get(url);
        return response.data;
      }

      // ids from the generate response
      const ids = `${data[0].id},${data[1].id}`;
      console.log(`ids: ${ids}`);

      for (let i = 0; i < 60; i++) {
        const result = await getAudioInformation(ids);
        if (result[0].status === "streaming") {
          console.log(`${result[0].id} ==> ${result[0].audio_url}`);
          console.log(`${result[1].id} ==> ${result[1].audio_url}`);
          break;
        }
        await new Promise((resolve) => setTimeout(resolve, 5000));
      }
      ```
    </CodeGroup>

    When `status` is `"streaming"`, the file is being written but is already playable. When it is `"complete"`, the full audio file is available.
  </Step>
</Steps>
